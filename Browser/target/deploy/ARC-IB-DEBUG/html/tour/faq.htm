Browser FAQ

                                                                                                                                       
Where is the full view tab? Browser does not allow a default Full View tab on versions. If you need one, ask your system administrator to explicitly define one in the version definition.
The function keys don't work. Why? Most browsers use the function keys for there own  functionality.
  • Use tab to move between fields on a contract.
  • Use return to commit a contract, search criteria or other web form.
How can I get Help? Clicking on the prompt in a version will show field level help. You can also double click on the input box for the field itself.
What major Desktop features are not supported?
 
 
 Only those features listed under the features list are available in 1.0 and proposed for 1.1. Major features that are not available include, but are not limited to:
  • Screen designer
  • Screen wizard
  • enquiry designer
  • enquiry wizard
  • report designer
  • crystal reports
  • end of day operations
What other features of Desktop are not supported? Minor features that are not supported include, but are not limited to:
  • Auto case changing on command line
  • User requests for other menus
  • Command line history stack (use browser auto complete options)

 
What development tool do we advise the customer to use, for developing new functionalities at client level? Browser is a packaged solution. There is no need to develop client side functionality. A all local developments should be done at the main tServer.
Is it advisable to develop new screens with a J2EE development studio and the Globus J2EE API? Standard versions, enquiries and menus should work well with Browser, though some minor formatting issues may need to be resolved by modifying the underlying version or enquiry - refer to the Browser User guide for details. Screen definitions remain at the server.
What is the plan for having complex character sets available? Korean? Traditional Chinese? Simplified Chinese?   Browser supports multi language character sets as standard in the same way that Desktop does. Full support for UTF8 is only available with jBASE 4.1
Is any application date stored at the web application server level? No. The only files that we store on the web server are images, static pages,  XSLT and css.
Which application servers are supported? Websphere 4.0.2/4.1, Weblogic 6.1
limited support for jBoss 3.0 and Oracle application server, full testing as part of phase 2
Oracle application server to be reviewed once it is fully JCA compliant.
What is the connectivity bean is
doing/providing and where does  it run?		 The connectivity bean is simply a way for us to abstract out the connection interface to the Connector. The Connector has two API's - the simple non J2EE api, and the more complex JCA api. The job of the connectivity bean is to use the correct API - as this implements an interface the calling objects don't need to know / care about the underlying architecture. All it does is take the message from the servlet and give it to the GC.
Do I need an application server such as websphere? You don't NEED the application server. We'll know more in terms of the scalability / performance issues after the benchmark in December 2002. We've run the whole setup with just using apache tomcat and it works well. It is a recommended and supported setup, depending on budget, scalibitly requirements, etc.
Do you offer an ASP model? No - the browser is servlet based, using completely dynamic HTML generated from XML messages transformed into HTML via XSLT and css.
Is it possible to use a web portal in which URLs would directly open Browser with specific functionality? Not in release 1.0, but this is proposed for 1.1via a web services architecture.
Does the web application server provide event information in case of problems to a monitoring tool, such as Tivoli?  Not in release 1.0, but this is proposed for 1.1, via the Event Service

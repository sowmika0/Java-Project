Browser Tour - What's changed


As you might expect, things are a little different in a browser as your connection  is now truly stateless. Some new approaches have been taken in the following areas:

  • Record locking
  • Field validation
  • Error handling
  • Overrides
  • Local Printing
Record Locking [back to top]

Browser performs record locking using a new table, RECORD.LOCK, that is used by both Desktop and Browser to claims locks on contracts. However, locks claimed by  Browser have an expiry time (set to your timeout as defined in you user profile. If you are not sure, ask you system administrator).

Field Validation [back to top]

Field validation does not get automatically carried out on every field. Instead, there are two mechanism to allow you to work in the same rich and interactive way that you are used to with Desktop. The Validate button will perform a full validation of the contract on request.

The other mechanism is by defining a field as a "hot field" in the version definition, meaning that when you leave the field the deal is submitted for validation.

Error Handling [back to top]

Errors are not reported against the fields, but instead are reported in a grid.

Overrides [back to top]

Overrides are now returned in a grid manner, displaying all of the overrides that require your approval.

Local Printing [back to top]

Local printing also works differently, as most browsers are not as tightly integrated with Windows as Desktop was. Hence, the instructions for printing will vary from browser to browser. However, local printing of enquiries, reports contracts and deal slips is supported.
